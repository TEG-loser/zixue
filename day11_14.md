# 一、最终代码：thread_pool.hpp（核心）
这是**工业级简化版线程池**，结构和 progschj/ThreadPool 完全一致。

```cpp
#ifndef THREAD_POOL_HPP
#define THREAD_POOL_HPP

#include <iostream>
#include <vector>
#include <queue>
#include <mutex>
#include <condition_variable>
#include <functional>
#include <thread>
#include <future>
#include <stdexcept>

class ThreadPool {
public:
    // 构造：启动 thread_num 个工作线程
    ThreadPool(size_t thread_num);

    // 析构：优雅退出
    ~ThreadPool();

    // 提交任务（核心接口）
    template<class F, class... Args>
    auto enqueue(F&& f, Args&&... args) 
        -> std::future<typename std::result_of<F(Args...)>::type>;

    // 禁用拷贝
    ThreadPool(const ThreadPool&) = delete;
    ThreadPool& operator=(const ThreadPool&) = delete;

private:
    // 工作线程列表
    std::vector<std::thread> workers_;
    
    // 任务队列
    std::queue<std::function<void()>> tasks_;
    
    // 同步
    std::mutex mutex_;
    std::condition_variable cv_;
    
    // 退出标志
    bool stop_;
};

// ===================== 实现 =====================
inline ThreadPool::ThreadPool(size_t thread_num) : stop_(false) {
    for (size_t i = 0; i < thread_num; ++i) {
        workers_.emplace_back([this] {
            while (true) {
                std::function<void()> task;

                {
                    std::unique_lock<std::mutex> lock(this->mutex_);
                    // 等待：停止 或 有任务
                    this->cv_.wait(lock, [this] {
                        return this->stop_ || !this->tasks_.empty();
                    });

                    // 优雅退出
                    if (this->stop_ && this->tasks_.empty())
                        return;

                    // 取任务
                    task = std::move(this->tasks_.front());
                    this->tasks_.pop();
                }

                // 执行任务（异常安全）
                try {
                    task();
                } catch (...) {
                    // 捕获异常，不崩溃线程池
                    std::cerr << "[ThreadPool] task exception caught\n";
                }
            }
        });
    }
}

// 提交任务
template<class F, class... Args>
auto ThreadPool::enqueue(F&& f, Args&&... args) 
    -> std::future<typename std::result_of<F(Args...)>::type> {
    
    using ReturnType = typename std::result_of<F(Args...)>::type;

    auto task = std::make_shared<std::packaged_task<ReturnType()>>(
        std::bind(std::forward<F>(f), std::forward<Args>(args)...)
    );

    std::future<ReturnType> res = task->get_future();

    {
        std::unique_lock<std::mutex> lock(mutex_);

        // 禁止向已停止的线程池提交任务
        if (stop_)
            throw std::runtime_error("enqueue on stopped ThreadPool");

        tasks_.emplace([task]() { (*task)(); });
    }

    cv_.notify_one(); // 唤醒一个线程
    return res;
}

// 析构 + 优雅退出
inline ThreadPool::~ThreadPool() {
    {
        std::unique_lock<std::mutex> lock(mutex_);
        stop_ = true;
    }
    cv_.notify_all(); // 唤醒所有线程

    // 等待所有线程结束
    for (std::thread& worker : workers_) {
        if (worker.joinable())
            worker.join();
    }
}

#endif
```

# 二、测试代码：main.cpp（验证功能）
```cpp
#include "thread_pool.hpp"
#include <iostream>
#include <chrono>

using namespace std;

int main() {
    // 创建 4 线程的线程池
    ThreadPool pool(4);

    // 提交 8 个任务
    vector<future<int>> results;

    for (int i = 0; i < 8; ++i) {
        results.emplace_back(
            pool.enqueue([i] {
                cout << "Task " << i << " running..." << endl;
                this_thread::sleep_for(chrono::seconds(1));
                return i * 10;
            })
        );
    }

    // 获取结果
    cout << "\nResults:\n";
    for (auto& res : results) {
        cout << res.get() << " ";
    }
    cout << "\n\nAll tasks done\n";

    return 0;
}
```

# 六、最终总结（可直接写进博客）
# 线程池总结
## 1. 核心思想
- 线程池 = **固定线程 + 任务队列 + 条件变量**
- 避免频繁创建/销毁线程
- 实现任务异步执行

## 2. 同步机制
- 互斥锁保护任务队列
- 条件变量实现**等待/唤醒**
- 无空轮询，CPU 利用率高

## 3. 关键设计
- `wait(lock, predicate)` 必须用条件判断
- 任务执行加 **try-catch** 保证异常安全
- 析构函数实现**优雅退出**
- 所有线程 join 后再退出

------
## 一、线程池整体架构
### 核心五大组件
1. **工作线程数组**：提前创建固定数量线程，常驻后台
2. **任务队列**：缓存待执行的异步任务
3. **互斥锁 mutex**：保护任务队列线程安全
4. **条件变量 cond**：实现线程**等待-唤醒**，避免空轮询
5. **停止标志 stop**：控制线程池优雅退出

### 核心工作模式
- 初始化：创建 N 个工作线程，全部阻塞在条件变量上
- 提交任务：外部把任务放入队列，唤醒一个工作线程
- 线程执行：工作线程抢任务、执行，无任务就阻塞等待
- 优雅退出：置停止标志 → 唤醒所有线程 → 等待所有线程收尾退出

---

## 二、必备知识点梳理
### 1. 多线程同步基础
- **互斥锁**：保护共享资源（任务队列），同一时刻仅一个线程访问
- **条件变量**：必须配合互斥锁使用；`wait` 会**自动解锁并阻塞**，被唤醒后**自动加锁**
- **虚假唤醒**：`cond_wait` 可能无信号也被唤醒，必须用 **while/谓词判断** 而不是 if

### 2. 线程池关键设计点
1. **任务抽象**：用 `std::function<void()>` 统一封装任意可执行任务
2. **异步返回**：`std::packaged_task + std::future` 支持任务带回返回值
3. **异常保护**：任务执行包 `try-catch`，单个任务崩溃不搞挂整个线程池
4. **优雅退出**：
   - 先置 `stop=true`
   - `notify_all` 唤醒所有阻塞线程
   - 逐个 `join` 等待工作线程正常结束
5. **禁止拷贝**：线程池持有线程、锁资源，禁用拷贝构造/赋值

### 3. 为什么要用线程池
- 避免频繁 `pthread_create/destroy` 开销
- 控制并发线程数，防止系统线程爆炸
- 任务队列解耦任务提交与执行
- 条件变量阻塞等待，**无 CPU 空轮询**

## 三、线程池极简伪代码
### 1. 线程池类结构伪代码
```plain
class ThreadPool {
  私有成员：
    线程数组 workers[]
    任务队列 queue<任务> tasks
    互斥锁 mutex
    条件变量 cond
    布尔 stop = false

  构造函数(线程数 n):
    循环 n 次：
      创建工作线程 执行 WorkerLoop

  析构函数:
    加锁 stop = true
    cond.notify_all()   // 唤醒所有线程
    遍历所有线程 join() // 等待全部退出

  提交任务接口 enqueue(任务):
    加锁
    如果 stop 抛出异常
    任务入队
    解锁
    cond.notify_one()   // 唤醒一个线程

  工作线程函数 WorkerLoop:
    死循环:
      加锁
      // 等待：有任务 或 要停止
      cond.wait(锁, 条件: stop 或 任务队列非空)
      
      // 优雅退出条件
      如果 stop 且 队列为空: 退出循环
      
      取出队首任务 出队
      解锁

      执行任务(try-catch 异常保护)
}
```

### 2. 工作线程核心逻辑伪代码
```plain
void WorkerLoop() {
    while (true) {
        加锁 mutex
        // 阻塞等待
        cond.wait(mutex, [&](){ return stop || !tasks.empty(); })

        // 退出判定
        if (stop && tasks.empty()) {
            退出线程循环
        }

        // 取任务
        task = tasks.front(); tasks.pop();
        解锁 mutex

        // 执行任务，捕获异常
        try {
            task();
        } catch (...) {
            日志捕获任务异常
        }
    }
}
```

## 四、完整执行流程
### 流程1：线程池初始化
1. 构造函数指定线程数量
2. 批量创建工作线程，每个线程进入 `WorkerLoop`
3. 所有工作线程立刻在 `cond_wait` 阻塞，**休眠等待任务**

### 流程2：外部提交任务
1. 调用 `enqueue` 提交任务
2. 加锁，任务推入任务队列
3. 解锁，调用 `cond.notify_one()`
4. 随机唤醒**一个**阻塞的工作线程

### 流程3：工作线程调度执行
1. 被唤醒后重新获取互斥锁
2. 检查条件：队列有任务
3. 从队列取出任务、出队、解锁
4. 执行任务，异常被内部捕获
5. 执行完回到循环，再次阻塞等待新任务

### 流程4：线程池优雅析构退出
1. 析构函数触发
2. 加锁设置 `stop = true`
3. `cond.notify_all()` 唤醒**所有**工作线程
4. 每个线程被唤醒后检测：`stop=true && 队列为空`
5. 工作线程逐个退出循环
6. 主线程 `join` 等待所有工作线程回收资源

---

## 五、关键易错点总结
1. **cond_wait 必须用谓词/while 循环**，防虚假唤醒
2. 锁的范围尽量小：**只保护队列操作，不保护任务执行**
3. 必须加任务异常捕获，防止单个任务崩掉工作线程
4. 停止标志必须配合锁保护，多线程可见性安全
5. 析构一定要 `notify_all + join`，否则线程泄露、程序卡死

---

## 六、一句话概括线程池原理
**提前创建固定工作线程阻塞在条件变量上，外部任务入队后唤醒线程消费任务，析构通过停止标志+广播唤醒实现优雅退出，用互斥锁保证任务队列线程安全。**

## 4. 适用场景
- 高并发短任务
- 网络服务器
- 异步任务处理

# Day22：fork 系统调用完整实验
包含：
- 基本创建子进程
- 父子进程区分（fork 返回值）
- 写时复制（Copy-on-Write）
- 全局变量/局部变量隔离实验
- 父子进程并发执行现象

## 最终成果：fork_test.cpp
```cpp
#include <iostream>
#include <unistd.h>
#include <sys/types.h>
#include <cstdio>
#include <cstring>

// 全局变量
int g_var = 100;

int main() {
    // 局部变量
    int l_var = 200;
    pid_t pid;

    printf("=== 开始 fork ===\n");
    printf("父进程 PID = %d，g_var = %d，l_var = %d\n\n", getpid(), g_var, l_var);

    // ================ 核心：fork 创建子进程 ================
    pid = fork();

    // fork 失败
    if (pid < 0) {
        perror("fork failed");
        return -1;
    }

    // ================ 子进程 ================
    if (pid == 0) {
        printf("【子进程】PID = %d，PPID = %d\n", getpid(), getppid());
        printf("【子进程】fork 返回值 = %d\n", pid);

        // 修改变量
        g_var = 999;
        l_var = 888;
        printf("【子进程】修改后：g_var = %d，l_var = %d\n\n", g_var, l_var);

        // 子进程退出
        _exit(0);
    }
    // ================ 父进程 ================
    else {
        // 等待子进程先执行（方便观察）
        sleep(1);

        printf("【父进程】PID = %d，子进程 PID = %d\n", getpid(), pid);
        printf("【父进程】fork 返回值 = %d\n", pid);

        // 关键：变量**没有被修改**，证明父子进程地址空间隔离
        printf("【父进程】变量未被修改：g_var = %d，l_var = %d\n\n", g_var, l_var);
    }

    printf("=== 进程结束 PID = %d ===\n", getpid());
    return 0;
}
```

# 一、fork 核心知识点（必须背）
## 1. fork 是什么？
- **系统调用**，用来**创建一个新的子进程**
- 子进程是父进程的**完整副本**（代码、数据、堆栈、文件描述符）
- 父子进程**完全独立**，有各自的 PID、地址空间
- 现代 Linux 使用 **写时复制（Copy-on-Write）** 优化性能

## 2. fork 的返回值（最重要）
- **= 0** → 运行在**子进程**
- **> 0** → 运行在**父进程**，返回值是**子进程 PID**
- **< 0** → 创建失败

## 3. 执行特点
- fork 之后，**父子进程同时运行**，调度顺序由操作系统决定
- 代码完全一样，通过返回值区分走不同分支
- 子进程会**继承**父进程打开的文件、信号处理、工作目录等

## 4. 地址空间隔离（实验重点）
- 子进程修改全局变量/局部变量
- **父进程完全不受影响**
- 证明：父子进程有**独立的虚拟地址空间**

# 二、fork 执行流程（伪代码）
```
主程序开始
打印父进程信息

调用 fork()
    ↓
内核创建子进程（复制页表，写时复制）
    ↓
两个进程同时从 fork 之后继续执行：

    子进程：返回 0 → 执行子进程代码
    父进程：返回 子PID → 执行父进程代码

父子进程各自结束
```

# 三、经典面试题
1. **fork 之后，父子进程共享什么？**
   不共享地址空间，共享：
   - 打开的文件描述符
   - 信号处理方式
   - 工作目录
   - 环境变量

2. **什么是写时复制？**
   fork 时不真正复制内存，只复制页表；
   只有**写入**时才复制物理页面 → 高效。

3. **子进程能看到父进程 fork 前的变量吗？**
   能看到初始值，但修改互不影响。

## 总结
- **fork = 创建子进程**
- **返回值 0 = 子进程**
- **返回值 >0 = 父进程**
- **父子进程地址空间完全隔离**（写时复制）
- 代码从 fork 之后**两份同时执行**

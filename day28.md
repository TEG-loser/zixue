# 第4周复盘：进程 vs 线程 + Linux IPC 方式完整总结
## 一、进程与线程核心对比
### 1. 基本定义
- **进程**：资源分配的最小单位，拥有**独立虚拟地址空间**、独立PID、完整资源集合。
- **线程**：CPU调度的最小单位，**共享所属进程地址空间**，仅有私有栈、寄存器。

### 2. 核心区别对照表
| 对比维度 | 进程 Process | 线程 Thread |
|--------|-------------|------------|
| 地址空间 | 完全独立，不共享 | 同进程内**完全共享**代码、全局变量、堆 |
| 资源归属 | 独占内存、文件描述符、信号上下文 | 共享进程资源，仅私有栈 |
| 创建开销 | 极大（复制页表、地址空间） | 极小（仅创建栈+TCB） |
| 通信方式 | 必须借助IPC：管道、信号、共享内存等 | 直接读写全局变量，只需加锁同步 |
| 容错性 | 一个进程崩溃不影响其他进程 | 一个线程崩溃**整个进程退出** |
| 调度粒度 | 粗粒度 | 细粒度，切换更快 |
| 数据共享 | 麻烦、需要IPC | 简单、天然共享 |

### 3. 联系
1. 一个进程至少包含**一个主线程**；
2. 线程依附进程存在，不能脱离进程独立运行；
3. 多线程适合**计算密集/高并发任务**，多进程适合**高隔离、高容错**场景。

### 4. 适用场景
- 多进程：服务器多进程模型、需要隔离稳定性、程序崩溃互不影响。
- 多线程：线程池、业务异步处理、高频任务调度、共享数据多的场景。

## 二、Linux 常见 IPC 方式对比复盘
本周学过的IPC：**匿名管道、信号**，扩展补充常用全套IPC，统一对比。

### 1. 各类 IPC 特点与适用场景
#### ① 匿名管道 pipe
- 特点：半双工、单向通信；**只支持有亲缘进程**（父子/兄弟）；内核环形缓冲、无磁盘IO。
- 同步：自带阻塞读写；
- 局限：无亲缘不能用、只能单向。

#### ② 信号 Signal
- 特点：**异步软中断**，内核/进程发给目标进程；
- 用途：通知事件（终止、闹钟、子进程退出）；
- 局限：携带数据少，适合简单通知，不适合传输大量数据。

#### ③ 命名管道 FIFO（拓展）
- 匿名管道升级；无亲缘进程也能通信；半双工、文件形式存在。

#### ④ 共享内存（拓展）
- 最快IPC；直接映射同一块物理内存到多个进程虚拟地址；
- 只负责共享，**需要额外加锁/信号量做同步**。

#### ⑤ 消息队列（拓展）
- 内核维护消息链表，按类型收发；支持离线消息。

### 2. IPC 方式横向对比表
| IPC方式 | 通信方向 | 适用进程关系 | 数据量 | 同步能力 | 特点 |
|--------|---------|-------------|--------|----------|------|
| 匿名管道 | 半双工 | 仅亲缘 | 中等 | 自带阻塞 | 简单、高效，只能父子 |
| 命名管道 | 半双工 | 无限制 | 中等 | 自带阻塞 | 跨进程、文件接口 |
| 信号 | 异步事件 | 无限制 | 极小 | 异步通知 | 适合事件通知，不传大数据 |
| 共享内存 | 双向 | 无限制 | 极大 | 无，需自行加锁 | 速度最快 |
| 消息队列 | 双向 | 无限制 | 中等 | 内核同步 | 按类型收发 |

### 3. 选型总结
1. **父子进程简单通信** → 匿名管道
2. **跨无关进程简单通信** → 命名管道
3. **事件通知、终止、定时** → 信号
4. **大批量高速数据传输** → 共享内存 + 互斥锁
5. **结构化消息、按类型分发** → 消息队列

# 三、fork 原理详解（核心重点）
## 1. 什么是 fork？
`fork()` 是 Linux **创建新进程**的唯一系统调用。
它会复制当前进程，产生一个**子进程**。

## 2. fork 执行流程
1. 内核为子进程分配 PID
2. 复制父进程**页表**（不复制物理内存）
3. 父子进程从 `fork()` 调用后**同时继续执行**

## 3. fork 返回值（最重要）
- **返回值 < 0**：创建失败
- **返回值 = 0**：当前处于**子进程**
- **返回值 > 0**：当前处于**父进程**，返回子进程 PID

## 4. 写时复制（Copy-On-Write）
现代 Linux 不直接复制内存，而是：
- 父子进程**共享同一份物理内存**
- 只有**写入时**才真正复制页面

优点：
- **极快**
- 避免无用内存拷贝
- 是 `fork + exec` 模型高效的基础

## 5. 父子进程关系
- 子进程继承：文件描述符、信号处理方式、工作目录、环境变量
- 子进程不继承：PID、未决信号、计时器
- 父子进程**完全独立，互不干扰**

# 四、exec 进程映像替换原理
## 1. exec 作用
用**新程序**覆盖当前进程的代码段、数据段、堆栈。
**进程 PID 不变**。

## 2. exec 执行逻辑
1. 清空原进程代码、数据
2. 加载新程序（如 `/bin/ls`）
3. 从新程序 `main` 开始执行

## 3. 为什么必须与 fork 配合？
- `fork` 创建独立进程
- `exec` 让子进程执行新程序

这是 **Linux 运行所有命令的底层方式**：
```
bash → fork → 子进程 → exec ls → 运行 ls
```

## 4. exec 家族（最常用）
```cpp
execlp("ls", "ls", "-l", NULL);
```
自动搜索 PATH，最实用。

---

# 五、管道（pipe）进程间通信
## 1. 管道是什么
- **内核缓冲区**
- 半双工（单向通信）
- 仅支持**亲缘进程**（父子/兄弟）

## 2. 管道结构
```
int fd[2];
fd[0] → 读端
fd[1] → 写端
```

## 3. 管道通信规则
- 写端写入 → 读端读出
- 空管道：读阻塞
- 满管道：写阻塞
- 所有写端关闭：读返回 0

## 4. 典型流程
```
父进程：pipe → fork → 关闭读端 → write
子进程：关闭写端 → read
```

## 5. 管道用途
- Shell 管道 `cmd1 | cmd2`
- 父子进程数据传递

---

# 六、信号（signal）异步通知机制
## 1. 信号是什么
**进程间异步软中断**。

## 2. 典型信号
- `SIGINT` (2)：Ctrl+C
- `SIGKILL` (9)：强杀，不可捕获
- `SIGCHLD` (17)：子进程退出
- `SIGALRM` (14)：定时器

## 3. 信号处理方式
1. 默认行为
2. 忽略 `SIG_IGN`
3. 自定义处理函数

## 4. 信号流程
```
进程注册 signal()
→ 内核发送信号
→ 进程中断当前流程
→ 执行信号函数
→ 恢复执行
```

## 5. 信号特点
- 异步
- 轻量
- 不能传大量数据
- 适合事件通知

# 七、简易 Shell 实现原理（Day26–27）
## 1. Shell 功能
1. 打印提示符
2. 读取命令
3. 解析命令
4. 执行命令（内置 + 外部）

## 2. 外部命令执行流程
```
父进程（Shell）
    ↓ fork()
子进程
    ↓ execvp()
执行命令（ls / cat / ps）
    ↓ 结束
父进程 wait() 回收
```

## 3. cd 为什么必须是内置命令？
- `cd` 必须修改**当前进程所在目录**
- 如果放在子进程执行，父进程不会改变
- 因此必须在 Shell 进程内直接调用 `chdir()`

## 4. Shell 主循环
```cpp
while (1) {
    打印提示符
    读取命令
    解析命令
    if (cd/exit)
        内置执行
    else
        fork + exec + wait
}
```

# 八、进程 vs 线程 深度对比（Step2）
## 1. 地址空间
- **进程**：独立虚拟地址空间，完全隔离
- **线程**：共享进程地址空间（代码、数据、堆、文件描述符）

## 2. 创建成本
- **进程**：高（复制页表、独立资源）
- **线程**：极低（仅创建栈 + 上下文）

## 3. 通信方式
- **进程**：必须用 IPC（管道/共享内存/信号）
- **线程**：直接共享全局变量，只需互斥锁

## 4. 稳定性
- **进程**：一个崩溃不影响其他
- **线程**：一个崩溃 → 整个进程退出

## 5. 适用场景
- **进程**：需要强隔离、稳定性高
- **线程**：高并发、任务频繁、需要共享数据

## 四、一句话核心总结
1. **进程是资源隔离单位，线程是调度执行单位；进程重隔离，线程重共享。**
